You don't need to close the code. You just need to own the road it runs on.

This week OpenAI [acquired Astral](https://astral.sh/blog/openai) — the company behind uv, ruff, and ty, the fastest-growing tools in the Python ecosystem. 126 million downloads per month. Three months earlier, Anthropic [acquired Bun](https://anthropic.com/news/anthropic-acquires-bun), the JavaScript runtime powering Claude Code. The declared reason is the same in both cases: integrate the feedback loop — build, lint, test, type-check — directly into the coding agent.

The code stays open. The license stays MIT. Nothing is locked. And yet something has shifted.

## The prosthesis becomes the world

Antonio Caronia [wrote in 2003](https://www.intfrancis.is/notes/caronia-prosthesis-world/) that the prosthetic paradigm — technology as an extension of the human body — breaks down with digital systems. Digital tools don't extend our reach into a stable world; they *constitute* the world the agent operates in. uv is not a screwdriver a developer picks up. It's the ground Codex walks on.

When OpenAI acquires Astral, it doesn't acquire a tool. It acquires a piece of the habitat. The enclosure doesn't need walls — it needs coordinates.

No version needs to be closed. An informational and temporal asymmetry is enough. Codex uses an internal build of uv optimized for its own patterns. The public version stays functional, one step behind. The facade remains open source. The advantage is structural.

## Fair use eats the commons

The pattern completes itself in copyright. In *Bartz v. Anthropic*, the court had already ruled that training on books is fair use. The FSF was notified because *Free as in Freedom* — Stallman's book, released under GFDL — was found in the training datasets. The FSF [noted](https://www.fsf.org/news/the-fsf-is-fighting-for-your-software-freedom-in-bartz-v-anthropic-inc): if copyleft applied, Anthropic would owe the release of weights, training data, and source code, not money.

But if training is not a derivative work, then copyleft does not propagate. The most copyleft book in existence gets absorbed into a proprietary system, and the legal mechanism designed to prevent exactly this does not fire. The commons is digestible. The enclosure is legally clean.

## The shape of the soft enclosure

Two cases, one structure. The code is open, the direction is captive — the fork is possible but the leadership work that made uv succeed where a dozen Python build tools failed is not forkable. Fair use neutralizes copyleft — the legal defense of openness becomes the legal path into the enclosure.

None of this requires malice. None of it requires closing anything. The enclosure is soft because it doesn't need to be hard. The code is free. The road belongs to someone.

Caronia, twenty-three years ago: "productive forces whose destructive character is inherent in the functioning of some software." Not an accident. The structure working as designed.
